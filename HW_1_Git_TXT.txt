TXT
 1. Создать внешний репозиторий c названием TXT.
находясь в Git Hub 
вверху слева в меню нажать new repository
записать название репозиотория TXT
проставить отметку public
проставить отметку Add a README file
нажать внизу Create repository

 2. Клонировать репозиторий TXT на локальный компьютер.
в Github войти в созданный репозиторий "TXT"
в вкладке "code" переключится на SSH и скопировать адрес репозитория  
в GitBash ввести команду git clone и вставить адрес внешнего репозитория TXT

 3. Внутри локального TXT создать файл “new.txt”.
touch new.txt

 4. Добавить файл под гит.
git sastus
git add 'new.json'

 5. Закоммитить файл.
git commit 'new file'

 6. Отправить файл на внешний GitHub репозиторий.
git push

 7. Отредактировать содержание файла “new.txt” - написать информацию о себе 
(ФИО, возраст, количество домашних животных, будущая желаемая зарплата).
 Всё написать в формате TXT.
cat >> new.txt
Бобкова Анна Александровна
35 лет
домашние животные - один рыжий кот
желаемая зарплата 1000 у.е.
ctrl+C

 8. Отправить изменения на внешний репозиторий.
git status
git commit 'new info'
git push

 9. Создать файл preferences.txt
touch preferences.txt

 10. В файл preferences.txt” добавить информацию о своих предпочтениях 
(Любимый фильм, любимый сериал, любимая еда, любимое время года,
 сторона которую хотели бы посетить) в формате TXT.
cat >> preferences.txt
Любимый фильм - Пятый элемент
Любимый сериал - Кухня
Любимая еда - гамбургер
любимое время года - лето
страна, которую хочу посетить - Италия
ctrl+C

 11. Создать файл sklls.txt добавить информацию о скиллах которые будут изучены на курсе в формате TXT
 touch sklls.txt

cat >> sklls.txt

 12. Сделать коммит в одну строку.
git status
git add .
git commit
'new file'

 13. Отправить сразу 2 файла на внешний репозиторий.
git push

 14. На веб интерфейсе создать файл bug_report.txt.
находясь в репозитории  TXT нажать Add file 
- create new file 
- ввести название файла - bug_report.txt
- нажать внизу commit new file

 15. Сделать Commit changes (сохранить) изменения на веб интерфейсе.
находясь в Github
открыть файл bug_report.txt
нажать Edit this file
написать текст
внизу заполнить commit changes, например new info
нажать на кнопку commit changes

 16. На веб интерфейсе модифицировать файл bug_report.txt, добавить баг репорт в формате TXT.
находясь в Github
открыть файл bug_report.json
нажать на иконку "карандаш" или Edit this file
написать баг репорт в формате txt -
Шаги к воспроизведению
1. Войдите в системы: Пользователь Тестер1, пароль xxxXXX
--> Вход в систему осуществлен
2. Кликните линк Профайл
--> Страница Профайл открылась
3. Введите Новое имя пользователя: Тестер2
4. Нажмите кнопку Сохранить
Результат
На экране появилась ошибка. Новое имя пользователя не было сохранено
Ожидаемый результат
Страница профайл перегрузилась. Новое значение имени пользователя сохранено. 

 17. Сделать Commit changes (сохранить) изменения на веб интерфейсе.
внизу заполнить commit changes, например new info
нажать на кнопку commit changes
